# 一、安全概览

## 模块定位

安全概览是安全中心的首页，用于集中展示当前系统整体安全状态。

它主要是展示和分析模块，不直接执行安全规则。

## 具体功能

### 1. 核心安全指标

展示：

```text
当前在线用户数
当前在线会话数
今日登录成功次数
今日登录失败次数
当前锁定账号数
今日触发限流次数
今日黑名单命中次数
今日风险事件数量
今日强制下线次数
待处理安全告警数量
```

### 2. 趋势分析

例如：

```text
近 7 天登录成功/失败趋势
账号锁定趋势
接口限流趋势
风险事件趋势
挑战验证趋势
在线用户趋势
强制下线趋势
```

### 3. 风险分布

按不同维度展示：

```text
风险等级分布
风险类型分布
风险应用分布
风险租户分布
风险 IP 分布
风险接口分布
```

### 4. 风险排行

例如：

```text
高风险账号排行
高风险 IP 排行
高风险设备排行
高风险接口排行
风险事件最多的应用
风险事件最多的租户
```

### 5. 实时安全动态

展示最近发生的：

```text
账号锁定
IP 封禁
异常登录
高风险访问
强制下线
策略变更
管理员解锁账号
安全告警
```

## 数据来源

安全概览的数据主要来自：

```text
安全事件
风险命中
处置执行记录
会话数据
安全告警
安全审计
```

## Nacos 降级

不部署安全中心时，可以不提供安全概览。

Nacos 只能配置是否记录相关事件，不能代替安全大盘的数据聚合能力。

# 二、凭证安全

## 模块定位

凭证安全负责管理用户用于证明身份的凭证。

第一阶段主要是密码，后续可以扩展：

```text
密码
PIN
API Key
Access Key
客户端密钥
恢复码
认证器密钥
```

当前你可以先以密码安全为主。

---

## 2.1 密码强度

### 功能

控制用户密码必须满足的规则：

```text
最小长度
最大长度
必须包含数字
必须包含大写字母
必须包含小写字母
必须包含特殊字符
禁止连续字符
禁止重复字符
禁止包含用户名
禁止包含手机号
禁止使用常见弱密码
```

例如：

```text
密码最少 10 位
必须包含字母和数字
不能包含用户名
不能使用 123456、password 等弱密码
```

### 触发场景

```text
用户注册
管理员创建用户
用户修改密码
管理员重置密码
首次登录修改密码
密码过期后修改密码
```

### 输出结果

```text
通过
不通过
不通过原因
密码强度等级
```

例如：

```json
{
  "passed": false,
  "violations": [
    "密码长度不能少于 10 位",
    "密码必须包含数字"
  ]
}
```

### 降级方式

完全可以通过 Nacos 配置并在本地执行。

---

## 2.2 密码有效期

### 功能

管理密码从设置完成后可以使用多长时间。

包括：

```text
密码有效天数
到期前提醒天数
密码过期后是否允许登录
过期后是否只允许访问修改密码接口
是否支持永不过期
```

例如：

```text
密码有效期 90 天
到期前 7 天提醒
过期后允许认证，但只能进入修改密码页面
```

### 需要的数据

用户级别需要保存：

```text
密码最后修改时间
密码过期时间
是否要求强制改密
```

### 和其他模块关系

密码过期后可能触发：

```text
凭证安全：判断密码已过期
处置管理：生成 FORCE_CHANGE_PASSWORD
会话安全：限制会话权限
安全事件：记录 PASSWORD_EXPIRED
```

### 降级方式

策略可以通过 Nacos 配置，用户密码时间需要存在数据库。

---

## 2.3 密码历史

### 功能

防止用户反复使用最近使用过的密码。

包括：

```text
禁止重复最近 N 次密码
历史密码保留数量
历史密码保留周期
管理员重置密码是否计入历史
```

例如：

```text
不能与最近 5 次密码相同
```

### 运行流程

用户修改密码时：

```text
验证新密码强度
对比历史密码
更新当前密码
写入密码历史
清理超出数量的历史记录
```

### 注意事项

密码历史表中保存的是密码哈希，不是明文密码。

### 降级方式

Nacos 配置历史数量，数据库负责保存历史数据。

---

## 2.4 初始密码

### 功能

管理管理员创建账号或重置密码后的初始凭证。

包括：

```text
是否生成随机初始密码
是否允许使用统一默认密码
首次登录是否强制修改
初始密码有效期
初始密码使用一次后是否失效
重置密码后是否强制下线旧会话
```

推荐企业级策略：

```text
使用随机初始密码
首次登录强制修改
初始密码短期有效
管理员无法查看用户修改后的密码
```

### 相关状态

用户可以有：

```text
initialPassword = true
forceChangePassword = true
```

### 降级方式

可以通过 Nacos 配置。

---

## 2.5 凭证风险

### 功能

这是凭证安全的高级能力，用于识别凭证是否可能已经不安全。

包括：

```text
弱密码风险
长期未修改密码
密码频繁重置
密码修改后快速登录失败
多个账号使用相同弱密码特征
疑似泄露密码
异常凭证使用
```

### 典型处置

```text
提醒修改密码
强制修改密码
要求 MFA
下线全部旧会话
锁定账号
发送安全告警
```

### 关系

凭证风险不是独立执行模块，通常输出给风险控制：

```text
凭证安全发现异常
→ 风险控制综合评分
→ 处置管理执行动作
```

### 降级方式

简单规则可以 Nacos 配置；泄露库、风险评分等复杂能力依赖安全中心。

---

# 三、账号保护

## 模块定位

账号保护负责保护账号本身，防止账号被爆破、盗用、恶意锁定或异常使用。

凭证安全关注“密码是否安全”，账号保护关注“账号当前是否还能正常使用”。

---

## 3.1 登录失败保护

### 功能

统计登录失败行为并执行保护措施。

支持维度：

```text
按账号统计
按 IP 统计
按设备统计
按 Client 统计
按账号 + IP 组合统计
```

配置项：

```text
统计时间窗口
最大失败次数
达到阈值后的动作
登录成功后是否清空
失败次数是否逐步衰减
```

例如：

```text
同一账号 5 分钟失败 5 次
→ 锁定账号 30 分钟

同一 IP 1 分钟失败 50 次
→ 封禁 IP 1 小时
```

### 运行数据

适合放 Redis：

```text
登录失败次数
第一次失败时间
最后一次失败时间
失败涉及账号数量
```

### 降级方式

可以使用 Nacos + Redis 完整降级。

---

## 3.2 账号锁定

### 功能

负责账号锁定状态的管理。

锁定来源可以是：

```text
登录失败自动锁定
管理员手动锁定
风险规则锁定
账号长期未使用锁定
凭证泄露锁定
合规要求锁定
```

锁定类型：

```text
临时锁定
永久锁定
自动锁定
人工锁定
```

记录内容：

```text
锁定原因
锁定时间
预计解锁时间
锁定来源
命中的规则
操作人
```

### 相关动作

账号锁定后通常需要：

```text
拒绝新登录
撤销 Refresh Token
强制下线已有会话
记录安全事件
必要时发送告警
```

### 降级方式

锁定阈值和时长可由 Nacos 配置；锁定状态需要 Redis 或数据库。

---

## 3.3 异常登录

### 功能

识别与用户正常登录习惯明显不同的行为。

包括：

```text
陌生设备登录
异地登录
异常时间登录
短时间跨地域登录
同一账号多 IP 并发登录
同一设备登录大量账号
管理员账号公网登录
长期未登录账号突然登录
```

### 处置方式

```text
仅记录
通知用户
要求验证码
要求 MFA
限制登录
拒绝登录
锁定账号
```

### 所需数据

```text
历史登录 IP
历史登录地点
历史登录设备
常用登录时间
常用客户端
```

### 降级方式

简单的陌生设备、并发登录阈值可以 Nacos 配置；行为画像和地域分析依赖中心服务。

---

## 3.4 账号状态

### 功能

统一管理账号在安全层面的状态。

不建议只用一个简单的 `status` 字段表示所有情况，推荐拆分：

```text
enabled
locked
expired
credentialsExpired
forceChangePassword
riskState
```

可能状态包括：

```text
正常
禁用
锁定
凭证过期
账号过期
要求改密
风险受限
待激活
```

### 作用

认证时需要综合判断：

```text
账号是否启用
账号是否锁定
账号是否过期
凭证是否过期
是否要求强制改密
```

### 降级方式

状态判断逻辑可以本地执行，但具体状态必须存储在数据库或 Redis。

---

# 四、访问防护

## 模块定位

访问防护负责保护系统入口、API 和网络访问，防止恶意调用、接口刷取、爬虫、爆破和非法来源访问。

执行位置主要是：

```text
Spring Cloud Gateway
ingot-auth
具体业务服务
```

---

## 4.1 黑白名单

### 功能

支持不同对象的黑白名单：

```text
IP 黑白名单
CIDR 网段黑白名单
用户黑白名单
设备黑白名单
Client 黑白名单
应用黑白名单
接口黑白名单
国家/地区黑白名单
```

名单类型可以是：

```text
永久
临时
自动生成
人工维护
风险规则生成
```

### 白名单作用

白名单不一定意味着完全放行，可以表示：

```text
跳过部分限流
允许访问特定接口
跳过验证码
仅允许内网 IP
允许可信设备
```

### 黑名单作用

```text
拒绝访问
限速
要求挑战验证
记录高风险事件
```

### 降级方式

小规模名单可通过 Nacos；大量动态名单适合 Redis 和安全中心数据库。

---

## 4.2 接口限流

### 功能

控制接口调用频率。

支持限流维度：

```text
全局
应用
Client
IP
用户
租户
接口
IP + 接口
用户 + 接口
```

常见算法：

```text
固定窗口
滑动窗口
令牌桶
漏桶
并发数限制
```

典型配置：

```text
登录接口：每 IP 每分钟 10 次
短信接口：每手机号每分钟 1 次
导出接口：每用户每小时 5 次
开放 API：每 Client 每秒 100 次
```

### 命中后动作

```text
返回 429
延迟响应
要求验证码
临时封禁
发送告警
```

### 降级方式

可以通过 Nacos 配置，执行必须在 Gateway 或本地服务完成。

---

## 4.3 防刷防爆破

### 功能

比普通限流更关注恶意行为模式。

包括：

```text
登录爆破
短信验证码刷取
邮箱验证码刷取
密码重置刷取
用户名枚举
接口遍历
爬虫抓取
批量账号测试
撞库攻击
```

### 与普通限流区别

普通限流是：

```text
请求次数超过阈值
```

防刷防爆破是：

```text
请求模式表现出攻击特征
```

例如：

```text
同一 IP 尝试 100 个不同账号
同一设备频繁更换手机号获取验证码
大量请求只改变用户 ID
```

### 降级方式

简单阈值可 Nacos 配置；复杂行为识别依赖风险控制和事件聚合。

---

## 4.4 网络访问控制

### 功能

控制什么网络来源可以访问系统或指定接口。

包括：

```text
仅内网访问
仅 VPN 访问
仅指定 IP 段访问
公网禁止访问
管理员后台来源限制
不同地区访问限制
代理/VPN/IDC IP 限制
请求真实 IP 识别
```

适用场景：

```text
Actuator 端点只能本地访问
管理员后台只允许公司网络
某些 API 只允许合作方固定 IP
某些接口禁止公网访问
```

### 降级方式

IP/CIDR 规则可以 Nacos 配置；代理识别、IP 风险库依赖外部数据或安全中心。

---

# 五、会话安全

## 模块定位

会话安全负责管理用户成功登录之后产生的登录态。

它管理的是：

```text
Session
Token
Refresh Token
sid
登录设备
在线状态
会话撤销
```

---

## 5.1 在线会话

### 功能

查看当前有效登录会话。

展示字段：

```text
用户
租户
应用
Client
sid
登录时间
最近访问时间
登录 IP
登录地点
设备
浏览器
操作系统
会话到期时间
会话状态
```

### 在线判断

推荐以 Redis 中的会话状态为准，不要只依赖 JWT 是否过期。

### 管理功能

```text
查看会话详情
查看用户全部会话
查看同 IP 会话
查看同设备会话
撤销指定会话
```

### 降级方式

Redis 可以继续维护会话；不部署安全中心时只是不提供统一管理页面。

---

## 5.2 登录设备

### 功能

管理用户用于登录的终端设备。

包括：

```text
设备标识
设备名称
操作系统
浏览器
首次登录时间
最近登录时间
是否可信设备
是否陌生设备
是否被禁用
```

可以支持：

```text
用户查看自己的登录设备
用户移除设备
管理员禁用设备
陌生设备登录提醒
可信设备免除部分挑战
```

### 降级方式

基本设备记录可以本地存储；可信设备统一管理需要安全中心支持。

---

## 5.3 并发会话

### 功能

控制同一用户允许同时存在多少个会话。

支持策略：

```text
允许无限会话
只允许一个会话
每个应用一个会话
每个设备一个会话
最多 N 个会话
管理员账号禁止并发登录
```

超限后的行为：

```text
拒绝新登录
踢出最旧会话
踢出全部旧会话
由用户选择踢出哪一个
```

### 降级方式

策略可以 Nacos 配置，执行依赖 Redis 会话索引。

---

## 5.4 强制下线

### 功能

主动撤销登录态。

支持范围：

```text
当前会话
指定会话
指定设备
某个用户全部会话
某个应用下的全部会话
某个租户全部会话
某个 IP 的风险会话
```

触发来源：

```text
管理员手动操作
用户主动退出
密码修改
密码重置
账号锁定
账号禁用
风险规则
权限重大变更
```

### 实现建议

JWT 中携带 `sid`，Redis 维护会话状态。

强制下线时：

```text
会话标记为 revoked
撤销 Refresh Token
必要时增加 token 黑名单
记录下线原因
```

### 降级方式

基础强制下线可通过 Redis 实现；跨应用集中治理更依赖安全中心。

---

# 六、挑战验证

## 模块定位

挑战验证用于在账号或请求存在一定风险时，追加身份确认。

它通常不是直接拒绝，而是：

> “你还需要再证明一次你是本人。”

---

## 6.1 验证码

### 功能

包括：

```text
图形验证码
滑块验证码
短信验证码
邮箱验证码
行为验证码
```

支持配置：

```text
验证码有效期
错误次数
发送频率
触发条件
使用场景
是否一次性
```

触发场景：

```text
登录失败次数过多
短信接口访问过快
密码重置
异常登录
高频操作
```

### 降级方式

触发策略可由 Nacos 配置；验证码状态需要 Redis。

---

## 6.2 MFA

### 功能

多因素认证，要求用户提供两种或以上不同类型的认证因素。

可以支持：

```text
TOTP
短信验证码
邮箱验证码
硬件安全密钥
Passkey
恢复码
```

管理功能：

```text
绑定 MFA
解绑 MFA
重置 MFA
恢复码生成
强制指定角色启用 MFA
管理员账号必须开启 MFA
```

### 触发模式

```text
每次登录都要求
新设备登录要求
异常登录要求
敏感操作要求
高风险事件要求
```

### 降级方式

Nacos 可以配置是否要求 MFA，但 MFA 密钥、绑定关系、恢复码不能只放 Nacos。

---

## 6.3 二次确认

### 功能

针对已经登录的用户，在执行敏感操作前再次确认身份。

方式包括：

```text
再次输入密码
输入 OTP
短信验证
邮箱验证
管理员复核
操作确认弹窗
```

适用场景：

```text
修改密码

导出敏感数据
分配管理员权限
删除重要数据
修改安全策略
生成访问密钥
```

### 与 MFA 区别

MFA 更多用于认证过程和高风险登录；二次确认更多用于已登录后的敏感操作。

### 降级方式

简单场景可以 Nacos 配置。

# 七、风险控制

## 模块定位

风险控制负责把来自不同安全域的信息组合起来，得出风险结论。

其他模块往往是单项判断：

```text
账号失败了几次
IP 请求快不快
设备是否陌生
密码是否过期
```

风险控制负责综合判断：

```text
当前行为整体风险有多高
是否允许继续
需要采取什么动作
```

---

## 7.1 风险规则

### 功能

定义什么条件属于风险。

规则可以包含：

```text
事件类型
匹配条件
统计维度
时间窗口
阈值
作用范围
风险等级
处置动作
```

例如：

```text
同一 IP 5 分钟登录失败超过 50 次
并且涉及 20 个以上账号
并且 IP 不在白名单
→ 严重风险
```

支持逻辑：

```text
AND
OR
NOT
次数聚合
去重计数
时间窗口
连续事件
```

### 降级方式

简单阈值规则可 Nacos 配置；复杂组合和规则编排依赖安全中心。

---

## 7.2 风险等级

### 功能

统一定义风险严重程度。

建议：

```text
LOW
MEDIUM
HIGH
CRITICAL
```

可以配置每个等级的默认动作：

```text
LOW      → 记录日志
MEDIUM   → 验证码
HIGH     → MFA、限流
CRITICAL → 拒绝、锁定、封禁、告警
```

风险等级避免不同模块各自定义“严重”“危险”等不统一概念。

### 降级方式

可以通过 Nacos 配置。

---

## 7.3 风险决策

### 功能

根据多个规则结果，输出最终安全决定。

典型返回：

```json
{
  "allowed": false,
  "riskLevel": "HIGH",
  "matchedRules": [
    "LOGIN_FAILURE_ACCOUNT",
    "UNKNOWN_DEVICE"
  ],
  "actions": [
    "REQUIRE_MFA"
  ]
}
```

风险决策要处理：

```text
多个规则同时命中
动作冲突
风险等级合并
白名单豁免
高优先级策略覆盖
```

例如同时命中：

```text
规则 A：要求验证码
规则 B：拒绝访问
```

最终应执行更严格的拒绝访问。

### 降级方式

简单本地决策可以执行；复杂评分和组合决策依赖中心化规则。

---

## 7.4 风险命中

### 功能

记录风险规则被触发的结果。

内容包括：

```text
命中的规则
风险等级
用户
IP
设备
应用
租户
请求接口
决策结果
执行动作
事件时间
```

### 与安全事件区别

安全事件是：

```text
发生了什么
```

风险命中是：

```text
某个事件为什么被判定为风险
```

例如：

```text
安全事件：LOGIN_FAILED
风险命中：LOGIN_FAILURE_IP_RULE
```

### 降级方式

不部署安全中心时可以写日志；查询和分析依赖安全中心。

# 十二、策略视图

## 模块定位

策略视图不是新的安全能力，也不是另一套策略数据。

它是把凭证安全、账号保护、访问防护等策略按照某个对象聚合展示。

例如进入某个应用，可以一次看到它所有生效的安全配置。

## 12.1 租户策略视图


# 十三、模块之间的完整运行关系

以一次登录为例：

```text
1. 访问防护
   检查 IP 黑白名单、登录接口频率

2. 账号保护
   检查账号是否锁定、登录失败次数

3. 凭证安全
   校验密码、判断密码是否过期

4. 风险控制
   综合账号、IP、设备、历史登录情况

5. 挑战验证
   根据风险决定是否要求验证码或 MFA

6. 会话安全
   登录成功后创建会话，检查并发会话限制

7. 处置管理
   执行锁定、下线、限流、告警等动作

8. 安全事件
   记录整个登录行为

9. 安全审计
   记录管理员干预和策略变更

10. 安全告警
    重大风险时通知安全人员
```

所以这些模块不是孤立的菜单，而是一条完整安全链路。

---

# 十四、建议的职责边界

| 模块 | 核心问题 |
|---|---|
| 凭证安全 | 用户使用的密码或凭证是否安全 |
| 账号保护 | 账号当前是否可以使用，是否正在被攻击 |
| 访问防护 | 当前请求是否来自允许来源，频率是否正常 |
| 会话安全 | 登录成功后的登录态是否安全、是否有效 |
| 挑战验证 | 是否需要用户再次证明身份 |
| 风险控制 | 综合各种信息后，风险有多高 |
| 处置管理 | 风险出现后具体执行什么动作 |
| 安全事件 | 系统发生了什么安全事实 |
| 安全审计 | 谁修改或操作了什么安全对象 |
| 安全告警 | 哪些安全问题需要通知和处理 |
| 策略视图 | 某个租户、应用、Client 或用户最终生效什么策略 |
| 安全概览 | 整个安全体系当前是什么状态 |

---

# 十五、你的第一阶段不需要全部做完

```text
└── 图形验证码

```

第二阶段再增加：

```text

```

第三阶段增加：

```text

```


最终最重要的设计原则是：

> **凭证、账号、访问、会话、挑战是基础安全能力；风险控制负责判断；处置管理负责执行；事件、审计、告警负责记录和运营；策略视图负责展示不同作用范围下最终生效的配置。**
