# Requirements

## 用户场景

### S1 密码即将过期与宽限期登录

- 使用者：任意已启用密码过期策略的用户（ADMIN / Member）。
- 触发条件：用户密码接近或超过 `maxDays` 有效期。
- 期望结果：
  - 进入警告窗口（`warningDaysBefore`）时，登录后能感知「密码即将过期，剩余 N 天」。
  - 过期后仍在宽限次数内：允许登录，但**每次成功登录扣减一次宽限次数**，并提示剩余次数。
  - 宽限次数耗尽后：登录被硬阻断（`CredentialsExpiredException`），或仅允许进入改密流程（按配置）。

### S2 管理员创建 / 重置账号后的初始密码

- 使用者：管理员创建用户或重置用户密码。
- 触发条件：`ADMIN_CREATE` 注册、`resetPassword`。
- 期望结果：
  - 支持两种初始密码来源：随机生成 / 统一默认密码（按策略二选一）。
  - 用户首次登录被要求强制改密（`mustChangePwd=true`）。
  - 初始密码短期有效（超期作废）、且改密成功后即失效（用后失效）。
  - 管理员无法查看用户改后的密码（现状已满足：只存哈希）。

### S3 强制改密引导

- 使用者：被标记 `mustChangePwd=true` / `force_change=1` 的用户。
- 触发条件：登录成功但存在强制改密标记。
- 期望结果：登录态可建立，但在完成改密前，仅能访问改密相关接口，其余接口被拒绝并引导改密。

### S4 降级模式下动态调整策略

- 使用者：运维 / 安全管理员。
- 触发条件：未部署安全中心（`ingot.security.credential.policy.mode=local`），需调整密码策略。
- 期望结果：修改 Nacos 中的密码策略配置，**无需重启**，新登录 / 改密即按新策略执行。

## 业务规则

1. 宽限期扣减仅在「密码已过期且仍在宽限内」的成功登录后发生；未过期或已锁定 / 禁用不扣减。（P0）
2. 宽限扣减必须与登录判定一致：判定「宽限内放行」与「扣减」使用同一份过期数据，避免并发下多扣或漏扣。（P0）
3. `force_change`（凭证域，`password_expiration`）与 `mustChangePwd`（账号域，`sys_user` / `member_user`）语义必须统一：任一置位都代表「必须改密」，改密成功后同时清除。（P0）
4. 初始密码策略可配置：`generation = RANDOM | FIXED`、`fixedPassword`、`validHours`（初始密码有效期）、`oneTime`（用后失效，默认 true）、`forceChangeOnFirstLogin`（默认 true）。（P1）
5. `ADMIN_CREATE` 与 `resetPassword` 不走密码强度校验（现状），但必须初始化 / 更新过期与 `force_change` 标记。（P1）
6. 强制改密拦截白名单：改密接口、登出、获取用户信息、必要的静态资源，其余拒绝。（P1）
7. Member 与 ADMIN 在启用相同策略时行为一致；未启用策略时保持 baseline（不阻断）。（P1）
8. `local` 模式下策略字段全部来自 Nacos 且动态刷新；`remote` 模式来自安全中心快照（现状已具备）。（P0）

## 边界与非目标

- 非目标：弱密码库 / 泄露密码检测、密码频繁重置风险评分（归凭证风险 2.5）。
- 边界：宽限期「过期后仅允许改密」与「宽限内正常登录」为两种不同配置语义，需在设计中区分（硬过期 vs 软过期）。
- 兼容：`password_expiration` 表结构不新增列（`force_change` 已存在），仅补齐领域模型映射与读写。
- 兼容：不改变现有 `local` / `remote` 双模式与 `RemoteCredentialService` 契约。
- Member 登录记录 / 自动锁定不在本次范围（L2 处理），本次仅确保 Member 凭证策略（过期 / 强制改密）生效。

## 验收标准

- [ ] 启用过期策略后，过期宽限内每次成功登录 `grace_login_remaining` 减一，耗尽后登录被阻断
- [ ] 警告窗口内登录返回「即将过期」提示，不阻断
- [ ] 管理员创建 / 重置账号后，用户首登被要求改密；初始密码超 `validHours` 失效；改密成功后初始密码即失效
- [ ] `force_change` 与 `mustChangePwd` 任一置位都要求改密，改密成功后同时清零
- [ ] 强制改密状态下访问非白名单接口被拒绝并引导改密
- [ ] Member 用户在启用过期 / 强制改密策略时行为与 ADMIN 一致
- [ ] `mode=local` 时修改 Nacos 密码策略配置，无需重启，新请求即生效
- [ ] 相关模块编译通过，已有登录 / 改密链路回归无破坏
