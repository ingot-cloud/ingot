## 目标

将 PMS 现有「应用通过根菜单和根权限隐式表达、菜单与权限通过 `org_type` 区分平台/租户」的模型，改造为以**应用**为资源归属与租户授权边界的明确架构。改造全程采用兼容迁移，不在单次发布中直接替换旧模型。

## 范围

### 包含

- 应用、菜单、权限的领域模型扩展与数据回填。
- 应用中心化资源管理接口与菜单托管权限生命周期。
- 精确权限 + 显式通配权限的新鉴权引擎。
- 租户应用授权、平台预设角色与租户自定义角色的统一接入。
- 菜单生成排序规则调整（根菜单先按应用排序，再按菜单排序）。
- 双模型影子对比、灰度切换与最终旧模型清理。

### 不包含

- 完整 ABAC 或策略语言。
- 菜单、权限和应用合并为单表。
- 部门数据权限模型重新设计。
- 前端增加应用一级导航。
- 为每个 HTTP 接口创建独立权限。
- 平台预设角色复制到每个租户。

## 设计原则

- 应用是资源归属和租户授权边界。
- 菜单负责导航，权限负责能力，二者都归属于应用。
- 菜单权限自动创建和同步，操作权限在应用权限树中维护。
- 精确权限只匹配自身，只有显式 `:*` 权限包含后代。
- 平台预设角色共享使用，租户只能追加权限，不能削减平台默认权限。
- 前端继续接收菜单树，根菜单先按应用排序，再按菜单排序。
- 全程采用兼容迁移，不在单次发布中直接替换旧模型。

## 实施顺序

阶段必须按顺序执行：

```text
0 → 1 → 2 → 3 → 4 → 5 → 6
```

进入下一阶段前，必须满足当前阶段的全部验收标准和退出条件。

阶段 0 至阶段 5 均需保留可回滚能力。阶段 6 只能在新模型稳定运行至少一个完整发布周期后执行。
